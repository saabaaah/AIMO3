# AIMO 3 - Evaluation & Submission Details

## Submission Format

### API-Based Submission
- Must use the provided Python evaluation API
- API serves test problems one-by-one in random order (public leaderboard)
- Fixed random order for private leaderboard
- Template: [AIMO 3 Submission Demo Notebook](https://www.kaggle.com/code/ryanholbrook/aimo-3-submission-demo)

### Notebook Requirements
- Submissions must be Kaggle notebooks
- Internet access MUST be disabled
- Submission file must be generated by the API
- Runtime obfuscation: up to 30 minutes variance in scoring time

### Runtime Limits
| Accelerator | Max Runtime |
|-------------|-------------|
| CPU Only | 9 hours |
| GPU | 5 hours |

## Scoring System

### Public Leaderboard (During Competition)
- Unnormalized accuracy (number of correct answers out of 50)
- Run on 50 public test problems only

### Private Leaderboard (Final Ranking)
Submissions are run TWICE on the 50 private test problems. Penalized accuracy scoring:

| Scenario | Score per Problem |
|----------|-------------------|
| Both predictions correct | 1.0 |
| One correct, one incorrect | 0.5 |
| Neither correct | 0.0 |

**Overall Score** = Sum of scores for all 50 problems (max = 50)

### Critical Notes
1. Submissions must complete successfully on BOTH private set reruns to be scored
2. Problems served in random order - must handle any ordering
3. 5-digit answers (0-99999) - no implicit modulo required
4. Any modulo calculation is explicitly stated in the problem

## Answer Format
- Integer between 0 and 99999 inclusive
- No automatic modulo conversion (unlike AIMO 1 & 2)
- May require basic computations including modular arithmetic
- Examples: 10! mod 997 = 265, 2^50 mod 1001 = 559

## Data Files
| File | Description |
|------|-------------|
| `reference.csv` | 10 reference problems with solutions |
| `test.csv` | 50 placeholder problems (replaced during scoring) |
| `sample_submission.csv` | Correct submission format |
| `AIMO3_Reference_Problems.pdf` | Full solutions to reference problems |

### CSV Fields
- `id`: Unique problem identifier
- `problem`: LaTeX problem statement
- `answer`: Integer 0-99999 (only in reference.csv)

## Private Set Rerun Process
1. During submission period: only public set (50 problems) is used
2. After deadline: all submissions rerun TWICE on private set (50 problems)
3. Predictions from both runs concatenated
4. Must complete successfully on both runs or submission is not scored

## Hardware Available
- **H100 GPUs**: Available exclusively for AIMO 3 notebooks (internet must be disabled)
- **Standard Kaggle GPUs**: P100, T4 options
- Significantly more compute than AIMO 2 (roughly double)
